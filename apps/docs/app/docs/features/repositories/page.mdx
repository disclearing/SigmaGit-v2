# Repositories

Repositories are the core of Sigmagit. Every project lives in a Git repository with full version control.

## Creating a Repository

From the web UI, click the **+** button in the header and select **New repository**.

### Via API

```bash
curl -X POST https://sigmagit.com/api/repositories \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "my-project",
    "description": "My awesome project",
    "visibility": "public"
  }'
```

## Cloning

### HTTPS

```bash
git clone https://sigmagit.com/username/repo.git
```

### SSH

```bash
git clone git@sigmagit.com:username/repo.git
```

## Repository Settings

Navigate to **Settings** in your repository to configure:

- **General** — Name, description, default branch
- **Visibility** — Public or private
- **Collaborators** — Add team members with roles
- **Danger Zone** — Archive or delete the repository

## Visibility

| Visibility | Who can view | Who can clone |
|-----------|-------------|--------------|
| Public | Everyone | Everyone |
| Private | Collaborators only | Collaborators only |

## Branches

Sigmagit supports all standard Git branching workflows.

### Branch Protection

Protect important branches (like `main`) from force-pushes and require pull request reviews before merging.

```bash
# Set default branch
git symbolic-ref HEAD refs/heads/main
```

## Forking

Fork any public repository to create your own copy:

1. Go to any repository
2. Click the **Fork** button
3. Choose your account or organization

After forking, your fork is fully independent with its own issues, PRs, and settings.

## Archiving

Archived repositories are read-only. They remain accessible but no new commits, issues, or PRs can be created.

<Callout type="info">
  Archiving is reversible — you can unarchive a repository at any time from Settings.
</Callout>
