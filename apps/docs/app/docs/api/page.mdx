# API Overview

Sigmagit provides a REST API for integrating with external tools and automating workflows.

## Base URL

```
https://your-sigmagit-instance.com/api
```

## Authentication

All API requests require authentication via a **Bearer token**.

### Creating a Token

1. Go to **Settings → Developer Settings → Personal Access Tokens**
2. Click **Generate new token**
3. Select the required scopes
4. Copy and store the token securely

### Using a Token

```bash
curl -H "Authorization: Bearer <your-token>" \
  https://sigmagit.com/api/user
```

### Token Scopes

| Scope | Description |
|-------|-------------|
| `repo` | Full control of repositories |
| `repo:read` | Read-only repository access |
| `issues` | Read and write issues |
| `pull_requests` | Read and write pull requests |
| `user` | Read user profile |

## Request Format

The API accepts and returns JSON. Always set the `Content-Type` header:

```bash
curl -X POST \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <token>" \
  -d '{"name": "my-repo"}' \
  https://sigmagit.com/api/repositories
```

## Response Format

Successful responses return `2xx` status codes with JSON bodies:

```json
{
  "id": "abc123",
  "name": "my-repo",
  "fullName": "username/my-repo",
  "description": "My repository",
  "visibility": "public",
  "createdAt": "2024-01-01T00:00:00Z"
}
```

## Error Responses

```json
{
  "error": "Repository not found",
  "code": "NOT_FOUND"
}
```

| Status | Meaning |
|--------|---------|
| `400` | Bad Request — invalid input |
| `401` | Unauthorized — missing or invalid token |
| `403` | Forbidden — insufficient permissions |
| `404` | Not Found |
| `409` | Conflict — resource already exists |
| `422` | Unprocessable Entity — validation error |
| `500` | Internal Server Error |

## Rate Limiting

The API is rate limited to **1000 requests per hour** per authenticated user. Rate limit headers are included in all responses:

```
X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 999
X-RateLimit-Reset: 1700000000
```

## Pagination

List endpoints return paginated results:

```bash
# Default: 30 items per page
GET /api/repos/owner/repo/issues

# Customize
GET /api/repos/owner/repo/issues?page=2&per_page=50
```

Response includes pagination metadata:

```json
{
  "items": [...],
  "total": 150,
  "page": 2,
  "perPage": 50,
  "totalPages": 3
}
```

## Common Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/api/user` | Get authenticated user |
| `GET` | `/api/repositories` | List user repositories |
| `POST` | `/api/repositories` | Create repository |
| `GET` | `/api/repos/:owner/:repo` | Get repository |
| `GET` | `/api/repos/:owner/:repo/issues` | List issues |
| `POST` | `/api/repos/:owner/:repo/issues` | Create issue |
| `GET` | `/api/repos/:owner/:repo/pulls` | List pull requests |
| `GET` | `/api/repos/:owner/:repo/commits` | List commits |
| `POST` | `/api/discord/link/generate` | Generate Discord link token |
| `POST` | `/api/discord/link/verify` | Verify Discord link token |
| `GET` | `/api/discord/link/status/:discordId` | Get Discord link status |
| `POST` | `/api/webhooks/discord` | Send Discord webhook |

See the detailed endpoint references for [Repositories](/docs/api/repositories), [Issues](/docs/api/issues), [Authentication](/docs/api/authentication), and [Discord](/docs/api/discord).
