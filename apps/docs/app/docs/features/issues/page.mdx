# Issues

Issues are the primary way to track bugs, feature requests, and tasks in Sigmagit.

## Creating an Issue

1. Go to your repository's **Issues** tab
2. Click **New issue**
3. Add a title and description
4. Optionally assign labels, assignees, and a milestone

### Via API

```bash
curl -X POST https://sigmagit.com/api/repos/username/repo/issues \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Bug: Login fails on mobile",
    "body": "Steps to reproduce...",
    "labels": ["bug", "mobile"]
  }'
```

## Issue States

| State | Description |
|-------|-------------|
| Open | Active, not yet resolved |
| Closed | Resolved or won't fix |

## Labels

Labels help categorize and filter issues. Each label has:

- A **name** (e.g., `bug`, `enhancement`, `documentation`)
- A **color** for visual identification
- An optional **description**

### Default Labels

Sigmagit creates these labels for new repositories:

| Label | Color | Description |
|-------|-------|-------------|
| `bug` | 🔴 Red | Something isn't working |
| `enhancement` | 🔵 Blue | New feature or request |
| `documentation` | 📚 | Improvements to docs |
| `question` | 🟣 | Further information requested |
| `help wanted` | 🟢 | Extra attention needed |
| `wontfix` | ⚪ | This will not be worked on |

## Assignees

Assign issues to team members to make ownership clear. An issue can have multiple assignees.

## Milestones

Group issues under a milestone to track progress toward a release or goal.

## Filtering & Search

Filter issues by:
- **State** — open / closed
- **Label** — filter by one or more labels
- **Assignee** — show only issues assigned to a person
- **Milestone** — filter by milestone

```
# URL filter examples
/issues?state=open&label=bug
/issues?assignee=username
/issues?milestone=v1.0
```

## Locking Issues

Maintainers can lock an issue to prevent further comments when a discussion becomes unproductive.

<Callout type="info">
  Locked issues are still visible but only collaborators with write access can comment.
</Callout>
