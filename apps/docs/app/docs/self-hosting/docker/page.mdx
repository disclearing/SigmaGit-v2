# Docker Deployment

The easiest way to deploy Sigmagit in production is with Docker Compose.

## Prerequisites

- Docker 24+
- Docker Compose v2+
- A domain name with DNS pointed to your server

## Docker Compose File

Create a `docker-compose.prod.yml`:

```yaml title="docker-compose.prod.yml"
version: '3.8'

services:
  web:
    image: ghcr.io/sigmagit/web:latest
    restart: unless-stopped
    environment:
      - API_URL=https://sigmagit.example.com/api
    ports:
      - "3000:3000"
    depends_on:
      - api

  api:
    image: ghcr.io/sigmagit/api:latest
    restart: unless-stopped
    environment:
      - DATABASE_URL=postgresql://sigmagit:password@db:5432/sigmagit
      - REDIS_URL=redis://redis:6379
      - S3_ENDPOINT=http://minio:9000
      - S3_BUCKET=sigmagit-repos
      - S3_ACCESS_KEY=${S3_ACCESS_KEY}
      - S3_SECRET_KEY=${S3_SECRET_KEY}
      - S3_REGION=us-east-1
      - S3_FORCE_PATH_STYLE=true
      - BETTER_AUTH_SECRET=${BETTER_AUTH_SECRET}
      - APP_URL=https://sigmagit.example.com
    ports:
      - "3001:3001"
    depends_on:
      db:
        condition: service_healthy
      redis:
        condition: service_started

  db:
    image: postgres:16-alpine
    restart: unless-stopped
    environment:
      - POSTGRES_DB=sigmagit
      - POSTGRES_USER=sigmagit
      - POSTGRES_PASSWORD=${POSTGRES_PASSWORD}
    volumes:
      - postgres_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U sigmagit"]
      interval: 5s
      timeout: 5s
      retries: 5

  redis:
    image: redis:7-alpine
    restart: unless-stopped
    volumes:
      - redis_data:/data
    command: redis-server --appendonly yes

  minio:
    image: minio/minio:latest
    restart: unless-stopped
    environment:
      - MINIO_ROOT_USER=${S3_ACCESS_KEY}
      - MINIO_ROOT_PASSWORD=${S3_SECRET_KEY}
    volumes:
      - minio_data:/data
    command: server /data --console-address ":9001"
    ports:
      - "9001:9001"  # MinIO console (remove in production)

volumes:
  postgres_data:
  redis_data:
  minio_data:
```

## Environment File

Create a `.env` file next to your compose file:

```bash title=".env"
POSTGRES_PASSWORD=your-secure-db-password
S3_ACCESS_KEY=your-minio-access-key
S3_SECRET_KEY=your-minio-secret-key
BETTER_AUTH_SECRET=your-32-char-secret
```

## Starting

```bash
# Pull latest images
docker compose -f docker-compose.prod.yml pull

# Start all services
docker compose -f docker-compose.prod.yml up -d

# Check status
docker compose -f docker-compose.prod.yml ps

# View logs
docker compose -f docker-compose.prod.yml logs -f api
```

## Initialize the Database

On first run, run migrations:

```bash
docker compose -f docker-compose.prod.yml exec api bun run db:migrate
```

## Updating

```bash
docker compose -f docker-compose.prod.yml pull
docker compose -f docker-compose.prod.yml up -d
```

## Backups

### Database Backup

```bash
docker compose exec db pg_dump -U sigmagit sigmagit | gzip > backup_$(date +%Y%m%d).sql.gz
```

### Restore

```bash
gunzip < backup_20240101.sql.gz | docker compose exec -T db psql -U sigmagit sigmagit
```

<Callout type="info">
  Set up automated backups with a cron job. Store backups off-site (S3, Backblaze B2, etc.).
</Callout>
