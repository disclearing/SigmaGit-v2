# Quick Start

Get Sigmagit up and running locally in minutes.

## Prerequisites

- [Bun](https://bun.sh) 1.3.5+
- [Docker](https://docker.com) (for PostgreSQL and Redis)
- [Git](https://git-scm.com)
- Node.js 20+ (optional, Bun includes a compatible runtime)

## 1. Clone the Repository

```bash
git clone https://github.com/sigmagit/sigmagit.git
cd sigmagit
```

## 2. Install Dependencies

```bash
bun install
```

## 3. Set Up Environment Variables

Copy the example environment file and fill in your values:

```bash
cp .env.example .env
```

At minimum, you need to configure:

```bash title=".env"
# Database
DATABASE_URL=postgresql://postgres:password@localhost:5432/sigmagit

# Redis (optional, for real-time features)
REDIS_URL=redis://localhost:6379

# S3-compatible storage for Git repos
S3_ENDPOINT=http://localhost:9000
S3_BUCKET=sigmagit-repos
S3_ACCESS_KEY=minioadmin
S3_SECRET_KEY=minioadmin
S3_REGION=us-east-1

# Auth secret (generate with: openssl rand -hex 32)
BETTER_AUTH_SECRET=your-secret-here

# App URL
APP_URL=http://localhost:3000
API_URL=http://localhost:3001
```

## 4. Start Infrastructure

Start PostgreSQL, Redis, and MinIO with Docker:

```bash
docker compose up -d
```

<Callout type="info">
  If you don't have a `docker-compose.yml`, you can start services individually or use existing PostgreSQL and Redis instances.
</Callout>

## 5. Set Up the Database

```bash
bun run db:push
```

## 6. Start the Development Server

```bash
bun run dev:web
```

This starts:
- **Web app** at [http://localhost:3000](http://localhost:3000)
- **API server** at [http://localhost:3001](http://localhost:3001)

## 7. Create Your First Account

Open [http://localhost:3000/register](http://localhost:3000/register) and create an admin account.
