# Development Setup

Welcome to the Sigmagit contributor guide! We love contributions of all kinds.

## Prerequisites

- [Bun](https://bun.sh) 1.3.5+
- [Docker](https://docker.com) (for local database/storage)
- [Git](https://git-scm.com) 2.40+
- A code editor (VS Code or Cursor recommended)

## Fork & Clone

```bash
# Fork the repo on sigmagit.com, then:
git clone git@sigmagit.com:YOUR_USERNAME/sigmagit.git
cd sigmagit
git remote add upstream git@sigmagit.com:sigmagit/sigmagit.git
```

## Install Dependencies

```bash
bun install
```

## Start Infrastructure

```bash
docker compose up -d
```

## Configure Environment

```bash
cp .env.example .env
# Edit .env — the defaults should work for local development
```

## Run Migrations

```bash
bun run db:push
```

## Start Development

```bash
# Start web app + API
bun run dev:web
```

The app will be available at [http://localhost:3000](http://localhost:3000).

## Project Structure

```
sigmagit/
├── apps/
│   ├── web/              # React app (TanStack Start + Vite)
│   │   ├── app/          # File-based routes
│   │   ├── components/   # UI components
│   │   └── lib/          # Utilities
│   ├── api/              # Hono API server
│   │   └── src/
│   │       ├── routes/   # API routes
│   │       ├── git/      # Git operations
│   │       └── auth.ts   # Auth config
│   ├── mobile/           # Expo app
│   └── discord-bot/      # Discord integration
└── packages/
    ├── db/               # Drizzle schema & migrations
    ├── hooks/            # Shared React Query hooks
    └── lib/              # Shared utilities
```

## Running Tests

```bash
cd apps/web && bun run test
```

## Making Changes

1. **Create a branch**: `git checkout -b feat/my-feature`
2. **Make your changes**
3. **Test your changes**: `bun run test`
4. **Commit**: `git commit -m "feat: add my feature"`
5. **Push**: `git push origin feat/my-feature`
6. **Open a Pull Request**

## Commit Message Convention

We follow [Conventional Commits](https://conventionalcommits.org):

```
feat: add new feature
fix: fix a bug
docs: update documentation
style: formatting changes
refactor: code refactoring
test: add or update tests
chore: maintenance tasks
```
