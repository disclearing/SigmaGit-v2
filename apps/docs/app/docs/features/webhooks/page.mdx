# Webhooks

Webhooks allow you to receive real-time notifications when events occur in your Sigmagit repositories. You can configure webhooks to send notifications to Discord, external services, or custom endpoints.

## Overview

Webhooks are HTTP callbacks that send POST requests to your configured URL when specific events occur. Sigmagit supports webhooks for:

- **Issues** — Opened, closed, edited, commented
- **Pull Requests** — Opened, closed, merged, reviewed
- **Pushes** — New commits pushed to repositories
- **Stars** — Repository starred or unstarred
- **Forks** — Repository forked

## Discord Webhooks

Sigmagit can send notifications directly to Discord channels via webhooks.

### Setup

1. **Create Discord Webhook**
   - Go to your Discord server
   - Navigate to Server Settings → Integrations → Webhooks
   - Click "New Webhook"
   - Choose the channel for notifications
   - Copy the webhook URL

2. **Configure Environment Variable**
   ```bash
   DISCORD_WEBHOOK_URL=https://discord.com/api/webhooks/YOUR_WEBHOOK_ID/YOUR_WEBHOOK_TOKEN
   ```

3. **Test Webhook**
   ```bash
   curl -X GET http://localhost:3001/api/webhooks/discord/test
   ```

### Webhook Payload Format

Discord webhooks receive JSON payloads with the following structure:

```json
{
  "type": "issue",
  "repository": {
    "owner": "username",
    "name": "repo-name"
  },
  "data": {
    "action": "opened",
    "number": 123,
    "title": "Issue title",
    "body": "Issue description",
    "state": "open",
    "author": {
      "username": "author",
      "name": "Author Name"
    },
    "labels": [
      { "name": "bug", "color": "d73a4a" }
    ],
    "createdAt": "2024-01-01T00:00:00Z"
  }
}
```

### Event Types

#### Issue Events

```json
{
  "type": "issue",
  "repository": { "owner": "user", "name": "repo" },
  "data": {
    "action": "opened",
    "number": 1,
    "title": "Bug in feature X",
    "body": "Description...",
    "state": "open",
    "author": { "username": "user", "name": "User Name" },
    "labels": [],
    "createdAt": "2024-01-01T00:00:00Z"
  }
}
```

**Actions:** `opened`, `closed`, `reopened`, `edited`

#### Pull Request Events

```json
{
  "type": "pull_request",
  "repository": { "owner": "user", "name": "repo" },
  "data": {
    "action": "opened",
    "number": 2,
    "title": "Add new feature",
    "body": "PR description...",
    "state": "open",
    "baseBranch": "main",
    "headBranch": "feature-branch",
    "author": { "username": "user", "name": "User Name" },
    "createdAt": "2024-01-01T00:00:00Z"
  }
}
```

**Actions:** `opened`, `closed`, `merged`, `reopened`

#### Push Events

```json
{
  "type": "push",
  "repository": { "owner": "user", "name": "repo" },
  "data": {
    "pusher": "username",
    "commits": [
      {
        "oid": "abc123...",
        "message": "Commit message",
        "author": { "name": "Author", "email": "author@example.com" }
      }
    ]
  }
}
```

#### Star Events

```json
{
  "type": "star",
  "repository": { "owner": "user", "name": "repo" },
  "data": {
    "action": "added",
    "username": "starrer"
  }
}
```

**Actions:** `added`, `removed`

#### Fork Events

```json
{
  "type": "fork",
  "repository": { "owner": "user", "name": "repo" },
  "data": {
    "username": "forker"
  }
}
```

## Custom Webhooks

You can also send webhooks to custom endpoints.

### API Endpoint

```
POST /api/webhooks/custom
```

### Request Body

```json
{
  "url": "https://your-endpoint.com/webhook",
  "events": ["issue", "pull_request", "push"],
  "secret": "your-webhook-secret"
}
```

### Webhook Delivery

When an event occurs, Sigmagit sends a POST request to your endpoint:

```http
POST https://your-endpoint.com/webhook
Content-Type: application/json
X-Sigmagit-Event: issue
X-Sigmagit-Signature: sha256=...

{
  "type": "issue",
  "repository": { ... },
  "data": { ... }
}
```

### Security

Webhooks include a signature header for verification:

```
X-Sigmagit-Signature: sha256=<hmac_sha256_signature>
```

Verify the signature:

```javascript
const crypto = require('crypto');

function verifySignature(payload, signature, secret) {
  const hmac = crypto.createHmac('sha256', secret);
  const digest = hmac.update(JSON.stringify(payload)).digest('hex');
  return crypto.timingSafeEqual(
    Buffer.from(signature),
    Buffer.from(`sha256=${digest}`)
  );
}
```

## Configuration

### Environment Variables

```bash
# Discord Webhook
DISCORD_WEBHOOK_URL=https://discord.com/api/webhooks/...

# Webhook Secret (for custom webhooks)
WEBHOOK_SECRET=your-secret-key

# Public Webhook URL (for external services)
PUBLIC_WEBHOOK_URL=https://your-domain.com/api/webhooks/discord
```

### Repository Settings

Configure webhooks per repository:

1. Go to repository Settings → Webhooks
2. Click "Add webhook"
3. Enter webhook URL
4. Select events to subscribe to
5. Save

## Rate Limiting

Webhook deliveries are rate-limited:

- **10 requests per second** per webhook URL
- Failed deliveries are retried with exponential backoff
- Maximum 3 retry attempts

## Troubleshooting

### Webhooks Not Delivering

- Check webhook URL is accessible
- Verify SSL certificate is valid
- Check firewall rules allow incoming connections
- Review server logs for errors

### Discord Webhook Not Working

- Verify webhook URL is correct
- Check webhook hasn't been deleted in Discord
- Ensure webhook channel still exists
- Test with `/api/webhooks/discord/test` endpoint

### Signature Verification Failing

- Ensure secret matches on both ends
- Check payload is not modified in transit
- Verify Content-Type header is `application/json`

## Best Practices

1. **Use HTTPS** — Always use HTTPS for webhook endpoints
2. **Verify Signatures** — Always verify webhook signatures
3. **Idempotency** — Handle duplicate deliveries gracefully
4. **Timeouts** — Respond quickly (under 5 seconds)
5. **Error Handling** — Log errors and handle failures
6. **Rate Limiting** — Implement rate limiting on your endpoint
