# Account Linking

Link your Discord account to your Sigmagit account to receive notifications, manage repositories, and access features directly from Discord.

## Overview

Account linking connects your Discord identity to your Sigmagit account, enabling:

- **Discord Notifications** — Receive repository notifications in Discord
- **Bot Commands** — Use Discord bot commands with your account context
- **Seamless Integration** — Access Sigmagit features from Discord

## Linking Methods

### Method 1: Discord Bot Command

1. Use `/link` command in Discord
2. Bot generates a verification token
3. Visit the Sigmagit website
4. Enter the token to complete linking

### Method 2: Email Verification

1. Use `/link-email [email]` command in Discord
2. Enter your Sigmagit account email
3. Bot generates a verification token
4. Check your email for verification link
5. Click the link to complete linking

## Linking Process

### Step 1: Generate Link Token

Use the Discord bot to generate a link token:

```
/link
```

Or with email:

```
/link-email email your@email.com
```

The bot will provide a verification token.

### Step 2: Verify Token

Visit your Sigmagit account settings and enter the token, or use the email verification link if you used the email method.

### Step 3: Complete Linking

Once verified, your accounts are linked. You can check status with:

```
/link-status
```

## Checking Link Status

Use the `/link-status` command to view:

- Whether your account is linked
- Verification status
- Linked Sigmagit username
- Link creation date

## Unlinking

To disconnect your Discord account:

```
/unlink
```

This removes the link between your Discord and Sigmagit accounts.

## API Endpoints

### Generate Link Token

```http
POST /api/discord/link/generate
Content-Type: application/json

{
  "discordId": "123456789012345678",
  "sigmagitEmail": "user@example.com"
}
```

**Response:**
```json
{
  "success": true,
  "token": "abc123..."
}
```

### Verify Link Token

```http
POST /api/discord/link/verify
Content-Type: application/json

{
  "token": "abc123...",
  "sigmagitUserId": "user-id"
}
```

**Response:**
```json
{
  "success": true,
  "linked": true,
  "user": {
    "id": "user-id",
    "username": "username",
    "email": "user@example.com"
  }
}
```

### Get Link Status

```http
GET /api/discord/link/status/:discordId
```

**Response:**
```json
{
  "linked": true,
  "verified": true,
  "user": {
    "id": "user-id",
    "username": "username",
    "name": "User Name",
    "avatarUrl": "https://..."
  },
  "linkedAt": "2024-01-01T00:00:00Z"
}
```

### Unlink Account

```http
POST /api/discord/link/unlink
Content-Type: application/json

{
  "discordId": "123456789012345678"
}
```

**Response:**
```json
{
  "success": true,
  "unlinked": true
}
```

### Relink Account

```http
POST /api/discord/link/relink
Content-Type: application/json

{
  "discordId": "123456789012345678",
  "sigmagitEmail": "user@example.com"
}
```

**Response:**
```json
{
  "success": true,
  "relinked": true,
  "user": { ... }
}
```

## Token Expiration

Link tokens expire after **1 hour** for security. If your token expires:

1. Generate a new token with `/link` or `/link-email`
2. Complete the verification process again

## Security

- Tokens are single-use and expire after 1 hour
- Tokens are cryptographically secure random strings
- Email verification ensures account ownership
- Unlinking requires Discord authentication

## Troubleshooting

### Token Expired

If your token expired, generate a new one:

```
/link
```

### Email Not Found

Ensure the email matches your Sigmagit account email exactly. Check for typos.

### Already Linked

If your account is already linked, you can:

- Check status: `/link-status`
- Unlink first: `/unlink`
- Then link again: `/link`

### Verification Failed

- Verify token hasn't expired
- Ensure token matches exactly (no extra spaces)
- Check API server is running and accessible
