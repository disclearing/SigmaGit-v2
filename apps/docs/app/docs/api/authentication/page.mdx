# Authentication

## Session-based Auth (Web)

The Sigmagit web app uses cookie-based sessions managed by [better-auth](https://www.better-auth.com). Login via:

- **Email & Password**
- **Passkey** (WebAuthn / FIDO2)

## Token-based Auth (API)

For API access, use **Personal Access Tokens (PATs)**.

### Creating a Token

```
Settings → Developer Settings → Personal Access Tokens → New Token
```

### Using a Token

```bash
curl -H "Authorization: Bearer sgp_yourtokenhere" \
  https://sigmagit.com/api/user
```

## Auth Endpoints

### Login

```http
POST /api/auth/sign-in/email
Content-Type: application/json

{
  "email": "user@example.com",
  "password": "your-password"
}
```

### Register

```http
POST /api/auth/sign-up/email
Content-Type: application/json

{
  "email": "user@example.com",
  "password": "your-password",
  "name": "Your Name",
  "username": "yourusername"
}
```

### Get Current User

```http
GET /api/user
Authorization: Bearer <token>
```

Response:

```json
{
  "id": "usr_abc123",
  "email": "user@example.com",
  "username": "johndoe",
  "name": "John Doe",
  "avatarUrl": "https://...",
  "createdAt": "2024-01-01T00:00:00Z"
}
```

### Logout

```http
POST /api/auth/sign-out
Authorization: Bearer <token>
```

## Passkeys (WebAuthn)

Sigmagit supports passwordless login via passkeys using the FIDO2 standard. Register a passkey in **Settings → Security → Passkeys**.

<Callout type="info">
  Passkeys are stored on-device and synced via your platform (iCloud Keychain, Google Password Manager, etc.). They are phishing-resistant.
</Callout>
