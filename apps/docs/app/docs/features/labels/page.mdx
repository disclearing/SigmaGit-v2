# Labels

Labels are colored tags you can apply to issues and pull requests to categorize them.

## Creating Labels

Go to **Issues → Labels** to manage labels for your repository.

Each label has:
- **Name** — A short identifier (e.g., `bug`, `v2.0`)
- **Color** — A hex color for visual identification
- **Description** — Optional context for what the label means

## Applying Labels

Labels can be applied when creating or editing an issue or PR. Multiple labels can be applied to the same item.

## Default Labels

New repositories come with a set of default labels:

| Label | Color | Description |
|-------|-------|-------------|
| `bug` | `#ef4444` | Something isn't working |
| `enhancement` | `#3b82f6` | New feature or request |
| `documentation` | `#8b5cf6` | Improvements or additions to docs |
| `duplicate` | `#6b7280` | This issue already exists |
| `good first issue` | `#22c55e` | Good for newcomers |
| `help wanted` | `#06b6d4` | Extra attention needed |
| `invalid` | `#f97316` | This doesn't seem right |
| `question` | `#eab308` | Further information requested |
| `wontfix` | `#6b7280` | This will not be worked on |

## Filtering by Label

Filter issues or PRs by label from the label filter dropdown, or via URL:

```
/issues?label=bug
/issues?label=bug&label=enhancement
```

<Callout type="info">
  Labels are shared between issues and pull requests in the same repository.
</Callout>
