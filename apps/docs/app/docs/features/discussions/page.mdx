# Discussions

Discussions provide a forum-like space for community conversations, Q&A, and announcements — separate from the issue tracker.

## Discussion Categories

Each repository can have custom categories:

| Category | Emoji | Best for |
|----------|-------|---------|
| General | 💬 | Open-ended conversations |
| Q&A | ❓ | Questions with accepted answers |
| Announcements | 📢 | Important updates |
| Ideas | 💡 | Feature brainstorming |
| Show & Tell | 🎉 | Showcasing projects |

## Answering Questions

In Q&A discussions, the original poster (and maintainers) can mark a comment as the **accepted answer**.

<Callout type="info">
  Q&A discussions with an accepted answer are marked with a ✅ checkmark in the list view.
</Callout>

## Pinning Discussions

Maintainers can pin up to 4 discussions to the top of the list for visibility.

## Locking Discussions

Lock a discussion to prevent new comments when a conversation has run its course.

## Converting Issues to Discussions

Sometimes an issue turns out to be a discussion-worthy topic. You can convert it from the issue's settings menu.
