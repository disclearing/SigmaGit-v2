# Installation

A detailed guide for installing Sigmagit in different environments.

## System Requirements

| Component | Minimum | Recommended |
|-----------|---------|-------------|
| CPU | 2 cores | 4+ cores |
| RAM | 2 GB | 8 GB |
| Disk | 20 GB | 100 GB+ |
| OS | Ubuntu 20.04+ | Ubuntu 22.04 LTS |
| Node / Bun | Bun 1.3.5+ | Latest stable |

## Installation Methods

### Docker (Recommended)

The fastest way to get started with a production-ready setup.

```bash
# Pull and start all services
docker compose -f docker-compose.prod.yml up -d
```

See the [Docker guide](/docs/self-hosting/docker) for the full compose file.

### Manual Installation

For environments where Docker is not available or for more control.

<Steps>

### Install Bun

```bash
curl -fsSL https://bun.sh/install | bash
```

### Clone the Repository

```bash
git clone https://github.com/sigmagit/sigmagit.git
cd sigmagit
bun install
```

### Configure Environment

```bash
cp .env.example .env
# Edit .env with your settings
```

### Build the Applications

```bash
bun run build
```

### Run Database Migrations

```bash
bun run db:migrate
```

### Start the Services

```bash
# Start API server
cd apps/api && bun run start &

# Start web server (serves from .output/server)
cd apps/web && bun run start &
```

</Steps>

## Database Setup

Sigmagit requires **PostgreSQL 14+**. See the [Database Setup guide](/docs/self-hosting/database) for detailed configuration.

```bash
# Create database
psql -U postgres -c "CREATE DATABASE sigmagit;"

# Run migrations
bun run db:migrate
```

## Storage Setup

Git repositories are stored in an S3-compatible object store. You can use:

- **AWS S3** — Production-ready, global availability
- **MinIO** — Self-hosted, compatible with S3 API
- **Cloudflare R2** — S3-compatible, no egress fees

See the [Storage guide](/docs/self-hosting/storage) for setup instructions.

## Reverse Proxy

For production, put Sigmagit behind a reverse proxy like Nginx or Caddy.

### Nginx Example

```nginx title="/etc/nginx/sites-available/sigmagit"
server {
    listen 80;
    server_name sigmagit.example.com;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl http2;
    server_name sigmagit.example.com;

    ssl_certificate /etc/letsencrypt/live/sigmagit.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/sigmagit.example.com/privkey.pem;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }

    location /api {
        proxy_pass http://localhost:3001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
    }
}
```

## Verifying Installation

After setup, check that everything is working:

```bash
# Check API health
curl http://localhost:3001/health

# Expected response:
# {"status":"ok","version":"0.1.0"}
```
